Generalized section

gsec œÑ œÑ' e e' means e' of oblivious type œÑ' is a section of e which has a public type œÑ.
Inductive gsec : expr -> expr -> expr -> expr -> Prop :=
| GSUnit e : gsec <{ ùüô }> <{ ùüô }> e <{ tape e }>

Generalized retraction

gret œÑ œÑ' e' e means e of public type œÑ is a retraction of e' which has an oblivious type œÑ'. Unlike gsec, retraction also allow the oblivious type œÑ' to contain non-oblivious components, as long as they match the corresponding components in œÑ.
Inductive gret : expr -> expr -> expr -> expr -> Prop :=
| GRId œÑ e : gret œÑ œÑ e e
| GRBool e' : gret <{ ùîπ }> <{ ~ùîπ }> e' <{ ~if tape e' then true else false }>
| GRProd œÑ1 œÑ1' œÑ2 œÑ2' e' e1 e2 :
    <{ œÑ1 * œÑ2 }> <> <{ œÑ1' * œÑ2' }> ->
    gret œÑ1 œÑ1' <{ œÄ1 e' }> e1 ->
    gret œÑ2 œÑ2' <{ œÄ2 e' }> e2 ->
    gret <{ œÑ1 * œÑ2 }> <{ œÑ1' * œÑ2' }> e' <{ (e1, e2) }>
| GRSum œÑ1 œÑ1' œÑ2 œÑ2' e' e1 e2 L1 L2 :
    <{ œÑ1 + œÑ2 }> <> <{ œÑ1' + œÑ2' }> ->
    (forall x, x ‚àâ L1 -> gret œÑ1 œÑ1' x <{ e1^x }>) ->
    (forall x, x ‚àâ L2 -> gret œÑ2 œÑ2' x <{ e2^x }>) ->
    gret <{ œÑ1 + œÑ2 }> <{ œÑ1' + œÑ2' }> e'
         <{ case e' of inl<œÑ1 + œÑ2> e1 | inr<œÑ1 + œÑ2> e2 }>

    (forall x, x ‚àâ L1 -> gret œÑ1 œÑ1' x <{ e1^x }>) ->
    (forall x, x ‚àâ L2 -> gret œÑ2 œÑ2' x <{ e2^x }>) ->
    gret <{ œÑ1 + œÑ2 }> <{ œÑ1' ~+ œÑ2' }> e'
         <{ ~case tape e' of inl<œÑ1 + œÑ2> e1 | inr<œÑ1 + œÑ2> e2 }>
(* Outsource to predefined context *)

Lifting

The core lifting relation. lift_core œÑ œÑ' e e' means e' of oblivious type œÑ' is the lifted result from e of public type œÑ.
Inductive lift_core : expr -> expr -> expr -> expr -> Prop :=
| LPi œÑ1 œÑ1' œÑ2 œÑ2' e e' r l' L1 L2 :
    (forall x, x ‚àâ L1 -> gret œÑ1 œÑ1' x <{ r^x }>) ->
    (* Types in lift_core are assumed to be non-dependent types, so œÑ2 and
    œÑ2' are locally closed. *)
    (forall x, x ‚àâ L2 -> lift_core œÑ2 œÑ2' <{ e (r^x) }> <{ e'^x }>) ->
    (* The function argument of e should have a leakage label of ‚ä§
    (LLeak), because e is applied to retraction. *)
    lift_core <{ Œ†:{‚ä§}œÑ1, œÑ2 }> <{ Œ†:{l'}œÑ1', œÑ2' }> e
              <{ \:{l'}œÑ1' => e' }>
| LSec œÑ œÑ' e e' :
    gsec œÑ œÑ' e e' ->
    lift_core œÑ œÑ' e e'
.

Top-level relation that handles type index arguments and calls lift_core to do the heavy lifting (pun intended). lift œÑs œÑ œÑ' e e' means e of type œÑ is lifted to e' of type œÑ' which has type indices œÑs.
Inductive lift : list expr -> expr -> expr -> expr -> expr -> Prop :=
| LCons œÑs œÑ œÑ1 œÑ2' e e' L :
    (forall x, x ‚àâ L -> lift œÑs œÑ <{ œÑ2'^x }> e <{ e'^x }>) ->
    lift (œÑ1::œÑs) œÑ <{ Œ†:œÑ1, œÑ2' }> e <{ \:œÑ1 => e' }>
| LNil œÑ œÑ' e e' : lift_core œÑ œÑ' e e' -> lift [] œÑ œÑ' e e'
.

Well-formedness of lifting input

œÑ is not a dependent type (at top level). It may still take as argument a dependently typed higher-order function.
Fixpoint nodep (œÑ : expr) : Prop :=
  match œÑ with
  | <{ Œ†:{_}_, œÑ2 }> =>
    lc œÑ2 /\ nodep œÑ2
  | _ => True
  end.

lift_type_wf œÑs œÑ' means œÑ' is a valid lifted oblivious type with indices œÑs. First, œÑs should be the "prefix" of œÑ', i.e. œÑ' is of the form Œ†:œÑ1, Œ†:œÑ2, ..., œÑ where œÑs is œÑ1, œÑ2, .... Second, index types in œÑs are locally closed, i.e. there is no dependent type in indices. Third, the type with indices instantiated is non-dependent, i.e. œÑ should have the form œÑ1 -> œÑ2 -> .... They can of course depend on the indices though.
Inductive lift_type_wf : list expr -> expr -> Prop :=
| LTVNil œÑ' : nodep œÑ' -> lift_type_wf [] œÑ'
| LTVCons œÑk œÑs œÑ' L :
    lc œÑk ->
    (forall x, x ‚àâ L -> lift_type_wf œÑs <{ œÑ'^x }>) ->
    lift_type_wf (œÑk::œÑs) <{ Œ†:œÑk, œÑ' }>
.

Equivalence up to revelation

in_range œÑ œÑ' v means value v of type œÑ is in the range of the oblivious type œÑ'. In other words, it is constrained by the indices in œÑ'.
Inductive in_range : expr -> expr -> expr -> Prop :=

| IRBool b : in_range <{ ùîπ }> <{ ~ùîπ }> b
| IRProd œÑ1 œÑ1' œÑ2 œÑ2' v1 v2 :

    <{ (gvar r) k v' }> ‚Üì v ->

We define the equivalence between a public expression and an oblivious expression up to revelation. The definition is essentially logical equivalence, but we do not treat higher-order arguments. The reason is rather subtle.
val_requiv œÑ œÑ' v v' means value v of public type œÑ and v' of (possibly) oblivious type œÑ' are equivalent up to revelation.
Inductive val_requiv : expr -> expr -> expr -> expr -> Prop :=
| VQRefl œÑ v : val_requiv œÑ œÑ v v
| VQBool b : val_requiv <{ ùîπ }> <{ ~ùîπ }> <{ b }> <{ [b] }>
| VQProd œÑ1 œÑ1' œÑ2 œÑ2' v1 v1' v2 v2' :

Section simple.

This handles the case when at least one of the expressions has simple type, i.e. not function types. The top-level equivalence is defined parametric in this relation, so we can instantiate it with different semantics or equi-termination later.
Variable expr_simple_requiv : expr -> expr -> expr -> expr -> Prop.

Expression e of public type œÑ and e' of oblivious type œÑ' are equivalent up to revelation.
Fixpoint expr_requiv_ (œÑ œÑ' : expr) (e e' : expr) : Prop :=
  match œÑ, œÑ' with
  | <{ Œ†:{_}œÑ1, œÑ2 }>, <{ Œ†:{_}œÑ1', œÑ2' }> =>
    forall v v',
      val v -> val v' ->
      ‚àÖ ‚ä¢ v :{‚ä•} œÑ1 ->
      ‚àÖ ‚ä¢ v' :{‚ä•} œÑ1' ->
      val_requiv œÑ1 œÑ1' v v' ->
      expr_requiv_ œÑ2 œÑ2' <{ e v }> <{ e' v' }>
  | _, _ => expr_simple_requiv œÑ œÑ' e e'
  end.

e of public type œÑ and e' of oblivious type œÑ' with indices œÑs are equivalent up to revelation. Essentially this says e and e' are equivalent expressions, i.e. expr_requiv, given any well-typed indices.
Fixpoint lift_requiv_ (œÑs : list expr) (œÑ œÑ' : expr) (e e' : expr) : Prop :=
  match œÑs with
  | _::œÑs =>
    match œÑ' with
    | <{ Œ†:{_}œÑ1, œÑ2 }> =>
      forall k : expr,
        ‚àÖ ‚ä¢ k :{‚ä•} œÑ1 ->
        val k ->
        lift_requiv_ œÑs œÑ <{ œÑ2^k }> e <{ e' k }>
    | _ => False
    end
  | [] => expr_requiv_ œÑ œÑ' e e'
  end.

End simple.

This is the definitional equivalence, using the small-step semantics. Note that we only consider partial equivalence here, i.e. we assume they both terminate. Equi-termination can be handled separately.
Definition expr_simple_requiv (œÑ œÑ' : expr) (e e' : expr) : Prop :=
  forall w w',
    e -->* w ->
    wval w ->
    in_range œÑ œÑ' (‚ü¶w‚üß) ->
    e' -->* w' ->
    wval w' ->
    val_requiv œÑ œÑ' (‚ü¶w‚üß) (‚ü¶w'‚üß).

Definition expr_requiv := expr_requiv_ expr_simple_requiv.
Definition lift_requiv := lift_requiv_ expr_simple_requiv.

      try auto_eapply; eauto using kinding;
        solve [ relax_typing_type; [ econstructor | ]; eauto ].
  (* GSSum *)

      simpl; simpl_cofin;

        | |- context [<{ ?œÑ1 +{_} ?œÑ2 }>] =>
          rewrite open_lc with (e := œÑ1) by eauto with lc;
            rewrite open_lc with (e := œÑ2) by eauto with lc
        end;
        econstructor; eauto using kinding, kinding_weakening_insert;
          auto_eapply;
          eauto using typing, kinding_weakening_insert with lc simpl_map.

                  | relax_typing_type; [ econstructor | ] ];

Stronger equivalence up to revelation

This version of equivalence uses the reveal semantics reval. It is stronger than the definitional version, and much more convenient for proofs because we do not have to deal with weak values and the wacky semantics.
Definition expr_simple_requiv_reval (œÑ œÑ' : expr) (e e' : expr) : Prop :=

    e ‚Üì v ->
